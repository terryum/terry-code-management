# Handoff Template — 다른 폴더로 작업 이전 시 사용

복잡한 잔여 작업은 해당 프로젝트 폴더에서 fresh 세션으로 진행하는 게 좋다. 사용자가 복사 붙여넣기로 바로 시작할 수 있도록 다음 형식으로 출력.

## 단일 핸드오프 블록 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[N] <repo-name> — <short task name>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📁 폴더:
   cd $CODES_ROOT/<group>/<repo>

📋 프롬프트 (그대로 복사 붙여넣기):
   ┌────────────────────────────────────────────────────
   │ <plan name 또는 task description>
   │
   │ 배경:
   │ - <key context line 1>
   │ - <key context line 2>
   │
   │ 작업:
   │ 1. <step 1>
   │ 2. <step 2>
   │ ...
   │
   │ 참고 파일:
   │ - <relative path 1>
   │ - <relative path 2>
   │
   │ 검증:
   │ - <verification 1>
   │
   │ 주의:
   │ - <caveat or risk if any>
   └────────────────────────────────────────────────────

🔗 관련 플랜: ~/.claude/plans/<plan-file>.md (있으면)
⏱  예상 소요: <짧음 / 중간 / 김>
🚦 상태: <Phase A 완료 / Phase B 진행 중 / 미착수>
```

## 작성 원칙

1. **`cd` 명령은 항상 absolute path** — 사용자가 어디에 있든 동작
2. **프롬프트는 self-contained** — 사용자가 새 세션을 시작해도 컨텍스트 부족 없도록 핵심 배경 1-3줄 포함
3. **참고 파일은 repo-relative path** (해당 폴더에서 작업하므로)
4. **주의사항은 짧고 구체적** — "force-push 필요", "supabase 마이그레이션 dry-run 먼저" 등
5. **여러 핸드오프가 있으면 우선순위 순으로** — 시급한 보안 / 영향 큰 것 / 간단한 것 순

## 특수 케이스

### 보안 / 긴급 작업
헤더에 🚨 표시 + 즉시 실행 권장 표시:
```
[1] 🚨 <repo-A> — <긴급 보안 이슈 한 줄> (긴급)
```

### 외부 액션 대기 중인 작업
프롬프트 대신 대기 사유와 트리거 조건만:
```
🟡 보류 — <외부 입력> 이 도착하면 plans/<plan-slug>.md 의 Step N 부터 실행
```

### 두 개 이상 repo 가 얽힌 작업
주 repo 폴더로 cd 시키되 다른 repo 도 명시:
```
📁 주 폴더: cd $CODES_ROOT/<group>/<repo-A>
   (참고: <repo-B>/scripts/ 가 심링크로 연결됨 — 거기에 변경이 들어감)
```
